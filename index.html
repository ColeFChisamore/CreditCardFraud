Introduction
This project was started and completed in October of 2025, and will not be updated any further. To see the entire project's files and updates on GitHub, please click HERE or use the link at the footer of the page.
In this project I am working with a downloaded public dataset on credit card claims (from Kaggle) to create a AI model to determine if a given claim is fraudulent. The dataset contains various pieces of information on 284,807 credit card claims such as:
  • Validity of claim: Either fraudulent (1) or not (0)
  • Dollar amount of claim: Integer representing the value of the claim
  • Time of claim: seconds since the very first claim recorded
  • V1 through V28: encrypted variables representing information that wouldn't be safe if made public
I will be working in Python in a Jupyter Notebook to manipulate the dataset and create my models. My models will be random forest classifier models from the Sci-Kit Learn library. This model works by making use of many tree models, which attempt to categorize information based on number of binary decisions. The random forest classifer creates many of these decision trees, then feeds a piece of information to all of them and returns the most commonly result from the trees. I will investigate the data along with my initial model to find any problems, then attempt to apply methods and create new models to fix them.

      Step 3.1: Preparing For the First Model
      First off, I need to import the machine learning libraries that I will be using to train this model. From the sklearn library (mentioned in step 1), I will import both the RandomForestClassifier and the train_test_split functions. The RandomForestClassifier is the AI classifier model itself that I will need to train, and the train_test_split is a function that will allow me to easily separate the training and testing data that I will feed to my model.
      But before I begin training, I need to set up the training data (used to train the model) and test data (used to evaluate the model). I have to separate the training and testing data to prevent the model from overfitting and memorizing this dataset rather than finding the underlying patterns. I begin by splitting the claims data into the explanatory x column and the response y column. Then, using the function I just imported, I split these columns into the training set and test set at a 4:1 ratio. This ratio is commonly used as a balance between having lots to train on while still keeping enough to test with afterwards.

            Step 4: Finding the Right Real:Fraud Claims Ratio
            As mentioned in step 2, the dataset has such a small number of fraudulent claims. It will be difficult for the model to get a low recall without adjusting the sampling. I will attempt to try different ratios of fraud:valid claims in sampling the X and Y for the training data, and see which ones give a good recall value without lowering precision or accuracy too far. I will do this by undersampling the valid claim, while maintaining every fraud claim.
            Step 4.1: A Quick First Sweep of Ratios
            To test which ratios work best I will go through many ratios from 1:1 up to 561:1 (basically the full set). I will increment evenly up by 20 to get an idea of how each statistic progresses as the ratio changes. I considered doing a logarithmic scale of increments to obtain finer detail at lower ratios, but decided against it since the statistics at that range are likely going to be quite poor anyways. At each step I will quickly train a new tree, use it to make predictions on the test set, and then measure and save various statistics in a dataframe. Basically, I am repeating step 3 for each of the ratios on the way. After completing each ratio, I save all of the data into a dataframe for ease of use.

This seems like the best recall-precision trade-off available, and it shows with the highest F1 score just barely (~0.01% higher than the full model).
